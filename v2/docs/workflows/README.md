# Workflows

This folder contains the step-by-step processes Claude follows for different types of work in this project.

> Claude must read the relevant workflow file **before** starting any work of that type.

---

## Available Workflows

| Workflow | File | When to use |
|----------|------|-------------|
| Analyse requirements | `analyse-requirements.md` | Turning requirements (Jira epic, FR list) into a spec + plan |
| Implement a ticket | `implement-ticket.md` | Picking up any sprint board task and building it |
| Debug | `debug.md` | Investigating a bug, test failure, or unexpected behaviour |

---

## How to Pick a Workflow

1. Identify the type of work being requested
2. Open the matching workflow file
3. Follow every step in order — no skipping

If no workflow exists for the task type, flag it to the human rather than improvising.
